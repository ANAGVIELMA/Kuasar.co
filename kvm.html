KVM (Kuasar Virtual Machines)

KVM (Kuasar Virtual Machines) es nuestro servicio IaaS altamente escalable y flexible con las capacidades que te facilitan desplegar máquinas virtuales en cuestión de minutos.

Con KVM sólo pagas por lo que usas, no requieres contratos ni permanencias que te aten para obtener descuentos, nuestros precios son bajos desde el momento que despliegas hasta cuando dejas de usar nuestra plataforma.

KVM está inicialmente disponible sólo en Bogotá, Colombia pero seguiremos expandiéndonos por la región. Debido a nuestra baja latencia puedes también usar este servicio para implementaciones de Edge Computing tales como streaming de video y IoT ya que estamos cerca a la fuente de datos y nos seguiremos acercando.

Actualmente te brindamos múltiples sistemas operativos OpenSource basados en Linux y propietarios como Windows.

KVM está comprendido por bloques que construyen el ecosistema cloud basado en máquinas virtuales. Haz clic en los enlaces abajo para obtener más información de cada bloque.

  • Cloud Privada Virtual

    Una red privada virtual o VPC es el contenedor que aisla los recursos de computo entre los diferentes usuarios del cloud de Kuasar.

    Proyectos

    Cuando creas una cuenta en Kuasar, el siguiente paso es crear al menos un proyecto. Un proyecto es un contenedor de recursos de cloud y de usuarios que puedes invitar a participar con diferentes roles asociados. Puedes crear tantos proyectos como requieras.

    VPC

    Es una red aislada y definida por un CIDR (Classless Inter-Domain Routing) privado, ejemplo, 172.16.0.0/16, donde un usuario puede crear subredes públicas y/o privadas dentro del mismo CIDR, un ejemplo de una subred puede ser 172.16.0.0/24, la cual debe estar contenida dentro del CIDR.
    Kuasar permite crear hasta 5 VPCs por proyecto y cada VPC puede contener máximo 6 subredes. Ni las VPCs ni las subredes tienen un costo. Si un usuario requiere definir más VPCs o subredes que las permitidas, se debe crear un ticket de soporte justificando su requerimiento.

    Subredes públicas y privadas

    En Kuasar definimos 2 tipos de subredes dentro de una VPC, subredes públicas y subredes privadas. Las subredes públicas son accesibles desde Internet mediante IP públicas con NAT 1a1 (una IP pública a una instancia), balanceadores de carga externos y PortForwarding. Adicionalmente debes configurar una ACL para que permita el tráfico a la subred.
    Las subredes privadas sólo permiten la creación de balanceadores internos no accesibles por fuera de la VPCs, por ende, no son viable las IP públicas, ni balanceadores externos ni Port Forwarding, estas subredes se usan para alojar servicios privados como bases de datos.
    Tantos las subredes publicas como las privadas pueden ser accedidas por el usuario o sitio remoto mediante el uso de VPN Remote access o VPN Site to Site respectivamente, las cuales se definen dentro de la VPC de Kuasar sin necesidad de componentes adicionales. Leer más

    Las VPNs son una forma segura de acceder a los recursos y tienen un costo asociado.

  • Acceso Remoto

    Kuasar brinda múltiples formas de conectarte a tu cloud de manera segura

    Gateway de Internet

    Una VPC en Kuasar ofrece por defecto un Gateway que permite salida a Internet tanto para subredes públicas como privadas a través de la técnica Source NAT, sin embargo, este acceso se puede restringir mediante la definición de una ACL asociada a una subred en particular.
    El Gateway de Internet ofrece servicio de DHCP a la VPC, si el usuario no define una IP para una máquina virtual puede obtener una IP privada de manera automática dentro la subred donde se definió.
    El Gateway de Internet también ofrece el servicio DNS que permite resolver nombres a IP dentro de la VPC y hacia internet. El servicio DNS registra el nombre de las máquinas creadas y permite su resolución dentro de la VPC.
    SEn Kuasar desplegamos estos Gateway de Internet de manera redundante para ofrecer mayor fiabilidad a nuestros usuarios.

    ACL

    Una ACL o Lista de Control de Acceso controla el tráfico dentro de una subred. Cada VPC tiene 2 ACLs por defecto, "permitir todo" y "denegar todo". Sin embargo, es posible crear nuevas ACL con permisos específicos. Cada subred dentro de una VPC puede tener asignada una ACL diferente o reusar la misma.

    Acceso Remoto

    VPN Site to Site

    Puedes crear una o varias VPNs IPsec desde una VPC existente a un sitio remoto, de esta forma podas ingresar directamente a administrar tu cloud.
    Cada túnel IPsec creado tiene un costo asociado. El límite de VPN permitidas por VPC es de 4.

    VPN L2TP Over IPSEC

    Adicionalmente puedes crear una VPN tipo L2TP Over IPsec por cada VPC. Esta VPN permite establecer conexión a usuarios móviles desde dispositivos basados en Linux, MacOS y Windows. El número máximo de usuarios permitidos son 8. Esta VPN tiene un valor asociado.

    IP Pública

    Para cada instancia creada dentro de una VPC, es viable asignar una dirección IP pública 1 a 1, lo cual se conoce como Static NAT, de esta forma y configurando la ACL, se puede permitir el acceso remoto a una máquina virtual desde una IP pública en Internet y desde ahí poder hacer conexión con el resto del ambiente. Sin embargo, esta forma es menos segura que las VPNs, por lo que en Kuasar configuramos nuestras instancias para que el usuario por defecto del sistema operativo se conecte únicamente con llaves SSH las cuales se deben crear previamente dentro del proyecto.

    En Kuasar ofrecemos direccionamiento IPv6 público por defecto. Para poder usar direcciones públicas IPv4 se debe crear un ticket con el requerimiento específico.

    Este método sólo se puede realizar para subredes públicas, en las subredes privadas no es posible configurar IP públicas, por lo que el usuario deberá tener al menos una subred pública, acceder a una instancia pública y luego conectarse al resto de instancias.

    Port Forwarding

    Otra forma de ingresar a las máquinas virtuales dentro de una VPC es mediante el método PortForwarding, donde se solicita una IP pública y sobre la misma se crean reglas de acceso a las diferentes máquinas virtuales basado en los puertos de destino, de esta forma una sólo IP pública serviría para acceder a una o varias instancias variando únicamente el puerto destino.

    Ejemplo, la IP1:2201 se puede usar para acceder a la máquina A vía SSH y la misma IP1:2202 a la máquina B por el mismo protocolo SSH. PortForwarding no tiene costo asociado dentro de la plataforma.

    Volúmenes

    Sólo disponemos para ti de volúmenes SSD NVMe que brindan la mayor velocidad, cada instancia cuenta con al menos un disco duro para el sistema operativo, pero es viable configurar más discos, los cuales puedes adjuntar a las instancias. Puedes desplegar la cantidad de discos que necesites.

    Todos los volumenes aprovisionados en Kuasar te brindan una alta disponibilidad, lo instalamos en hardware diferente a las instancias y los replicamos en nuestra infraestructura para que nunca pierdas información.

    El disco root o principal se crea y se elimina con la máquina virtual, los discos adicionales adjuntados o también conocidos como discos de datos, permanecen disponibles luego de eliminar la instancia a la cual pertenecían, por lo que viable adjuntarlos a otra.

    El tamaño máximo posible de un volumen es 1TiB, si se requiere más, se aconseja adjuntar varios discos y configurarlo en volúmenes lógicos dentro del sistema operativo para obtener particiones más grandes.

    Snapshots

    Los snapshots son respaldos de los volúmenes y tienen como propósito proteger la información almacenada, los snapshots se almacenan en discos más lentos por lo que el costo por GB/mes final es menor.

    Con los snapshots puedes crear nuevos volumenes y adjuntarlo a instancias, adicionalmente puedes generar imagenes booteables para desplegar nuevas instancias, esto es especialmente útil cuando tienes aplicaciones con múltiples nodos.

    Imágenes

    Las imágenes son plantillas que se pueden crear desde los snapshots o desde los volumenes para crear nuevas instancias. Existen 2 tipos de imagenes en Kuasar, las creadas por nosotros para que puedas lanzar sistemas operativos al instante o las creadas por ti para reusarlas y compartirlas entre tus proyectos o con otros usuarios de nuestra plataforma.

    Despliega instancias de manera fácil, ágil y flexible mediante un par de clics o de manera programática con nuestras herramientas Devops que te permitirán automatizar tu infraestructura. Conoce nuestros precios aquí.

    Múltiples funcionalidades
    DHCP

    Por defecto nuestra VPC le configura una IP a tu instancia dentro de la subred donde la incluiste al momento de su creación, o si lo prefieres le puedes definir la ip que requieras pero debes tener en cuenta que debe estar dentro CIDR de la VPC y dentro de la subred que le asignas.

    DNS

    Adicionalmente podrás desplegar instancias con tu propio dominio y comunicarte por defecto con tus otras instancias dentro de la misma VPC mediante el nombre que le configuraste en el momento de la creación, no necesitas saber la IP.

    IP Adicionales

    Una vez lanzada la instancia puedes configurarle a tu interfaz de red, ip secundarias dentro de nuestra plataforma, con esto te damos flexibilidad para tus despliegues más exigentes.

    Interfaces adicionales

    ¿Necesitas interfaces en subredes diferentes?, es posible, adiciona las que necesitas dentro de la subred que requieras.

    Llaves SSH

    Nuestras imagenes Linux vienen configuradas por defecto para no aceptar contraseñas de acceso para el usuario principal, por lo que deberás crear o cargar la llave publica siempre antes de lanzar una instancia, de esta forma te podrás conectar via SSH para administrarla.
